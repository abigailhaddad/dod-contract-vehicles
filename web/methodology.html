What this dashboard shows

Active DoD contract vehicles — IDIQs, BPAs, GWACs, and other awards — with scope, ceiling, obligations, and remaining capacity. Built to help program offices find existing vehicles to place orders on instead of starting a new procurement.

  • Contracts — one row per contract award (task order, call, or standalone).
  • Vehicles — orders rolled up by parent_award_id_piid, so one row per parent IDIQ / BPA.

Data sources

  • USASpending bulk archives — transaction-level contract records, full column set.
  • SAM.gov monthly entity extract — supplements contractor details (UEI, legal name, registration date, location).

Scope: toptier agency — (DoD), last — fiscal years of transactions, rebuilt monthly.

Taking the max across these fields gets us the most generous true end date.

Buckets

  • Active — effective end is more than — days out.
  • Expiring Soon — effective end within the next — days.
  • Expired — effective end before today.
  • Unknown — no parseable date in any of the effective-end fields.

Drop policy

  • Drop Expired: —
  • Drop Unknown: —

By default we drop both. An expired contract's ordering period has lapsed, so you can't place new orders against it. An Unknown contract has no end date in the source and can't be verified without going back to SAM.gov anyway — so it's noise for this use case.

Vehicle rollup

The Vehicles view groups contracts by parent_award_id_piid. For each vehicle:

  • Status — classified from the latest effective end across its orders.
  • Total Ceiling / Obligated — summed across orders; not the IDV's own ceiling field.
  • Active Orders — count of task orders currently Active or Expiring Soon.
  • Contractors / NAICS / States — unique values seen across orders.
Caveat on ceiling rollup: summing order-level ceilings can overcount (if options share a ceiling) or undercount (if the IDV ceiling hasn't been fully ordered against). Treat the totals as approximate.

Other caveats

  • ceiling_remaining = potential_total_value_of_award − total_dollars_obligated. USASpending's potential_total_value_of_award reflects the latest modification and may lag real ceiling changes.
  • award_description is often terse ("IT SERVICES"). For real scope, check the solicitation on SAM.gov.
  • Geographic coverage is place-of-performance state, not the vehicle's authorized geographic scope.
  • Aggregation keeps only the latest transaction's values per contract (by action_date); federal_action_obligation is summed across all mods as a fallback when total_dollars_obligated is missing.
